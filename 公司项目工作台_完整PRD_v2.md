# 公司项目工作台完整 PRD v2

## 1. 产品概述

公司项目工作台是一套面向公司内部的项目推进、岗位协作、会员权益兑付、活动签到与素材沉淀系统。

系统围绕公司经常举办的课程、活动、招商项目展开，支持多个项目并行推进。总控或项目负责人负责创建项目、发布任务、分发到岗位；各岗位员工在自己的工作台接收任务、执行任务、上传成果、更新状态；项目总览台用于查看整体进度、到场确认、风险、会员权益兑付和周例会汇报。

原有「人物素材库」能力保留，但不再作为系统唯一入口，而是归入「素材库 > 人物素材」。人物资料同时承载客户/会员基础资料、权益台账、参与记录和待兑付信息。

## 2. 产品目标

- 支持多个课程/活动项目并行推进。
- 总控统一创建项目并向各岗位发布任务。
- 岗位人员只处理被分发到自己的任务。
- 管理者可在总览台查看项目进度、风险、招商确认名单、签到情况和权益兑付情况。
- 设计、现场等岗位上传图片后，自动沉淀到项目图片库。
- 项目图片库只保存图片类型素材，默认展示最新版，避免误用旧图。
- 人物素材库升级为人物/客户/会员资料中心。
- 支持自定义会员政策和权益内容。
- 支持客服进行权益兑付操作。
- 支持活动扫码签到，老客户自动匹配，新客户自动创建临时档案。
- 支持 AI 一键生成周例会 HTML 可视化报表。

## 3. 角色与权限

### 3.1 总控 / 项目负责人

总控是项目的创建者、任务分发者和进度监管者。

主要权限：
- 创建项目。
- 设置项目基本信息。
- 发布任务并分发到岗位。
- 设置任务负责人、截止时间、优先级、成果要求。
- 查看项目总览。
- 查看招商确认名单。
- 查看签到名单。
- 查看风险任务。
- 审核关键成果。
- 查看会员权益兑付进度。
- 生成 AI 周例会报告。

### 3.2 招商人员

招商人员负责邀约、确认参加、跟进客户和查看可邀约会员。

主要权限：
- 查看被分发给招商岗位的任务。
- 查看当前项目可邀约会员名单。
- 查看会员剩余权益。
- 标记客户确认参加。
- 取消确认参加。
- 添加邀约备注。
- 查看项目图片库中的最新版宣传图。
- 查看自己名下的确认到场名单、已签到名单、未到场名单。

招商人员不负责正式核销权益。

### 3.3 客服人员

客服负责报名确认、课前提醒、资料完善、会员权益兑付和核销确认。

主要权限：
- 查看客服岗位任务。
- 查看权益兑付待办。
- 标记已通知、已兑付、延后、放弃。
- 处理扫码签到产生的新客临时档案。
- 完善客户资料。
- 查看项目图片库中的最新版海报、二维码图。

### 3.4 市场中心岗位人员

市场中心包括：
- 设计
- 文案
- 短视频
- 运营
- 客服
- 现场执行

各岗位只看到总控分发给自己的任务。

### 3.5 管理者

管理者主要使用项目总览台和 AI 周报。

主要关注：
- 项目整体进度。
- 各岗位任务完成情况。
- 招商确认人数。
- 到场签到人数。
- 风险任务。
- 待审核成果。
- 权益兑付情况。
- 本周工作总结和下周重点。

## 5. 全局项目机制

### 5.1 并行项目

系统支持多个项目同时推进，例如：

- 众创截拳道 · 2026年5月课程
- 众创截拳道 · 2026年6月课程
- 招商说明会 · 5月场
- 私董会沙龙 · 6月场

每个项目都包含：
- 项目系列
- 项目名称
- 活动日期
- 项目阶段
- 项目负责人
- 项目目标
- 所属任务
- 所属图片
- 招商名单
- 签到记录
- 权益兑付记录

### 5.2 当前项目简介栏

所有工作台页面顶部都有当前项目简介栏。

默认展示距离当前日期最近的进行中项目。

展示内容：
- 当前项目名称
- 所属系列
- 活动日期
- 距离开始天数
- 当前阶段
- 项目负责人
- 整体完成度
- 风险任务数
- 切换项目按钮

切换项目后，以下内容联动切换：
- 项目总览
- 招商中心
- 各岗位工作台
- 项目图片库
- 权益兑付列表
- 签到名单
- AI 周报数据范围

## 6. 项目总控

### 6.1 创建项目

入口：`项目总控 > 创建项目`

字段：
- 项目系列
- 项目名称
- 活动日期
- 项目阶段
- 项目负责人
- 招商目标
- 到场目标
- 成交目标
- 备注说明

示例：

```text
项目系列：众创截拳道
项目名称：2026年5月课程
活动日期：2026-05-31
阶段：集中邀约 / 市场预热
目标：报名80人、到场60人、成交20人
```

创建项目后，进入任务分发页面。

### 6.2 发布任务

总控给岗位发布任务。

字段：
- 所属项目
- 所属岗位
- 任务名称
- 任务说明
- 负责人
- 截止时间
- 优先级
- 是否需要上传成果
- 成果类型
- 是否需要审核
- 是否关键节点
- 关联素材

岗位选项：
- 招商
- 设计
- 文案
- 短视频
- 运营
- 客服
- 现场执行

成果类型：
- 图片
- 文案
- 表格
- 视频
- 其他

发布后，任务自动进入对应岗位工作台。

### 6.3 任务分发看板

按岗位分列展示：
- 招商
- 设计
- 文案
- 短视频
- 运营
- 客服
- 现场执行

任务卡字段：
- 任务名称
- 负责人
- 截止时间
- 状态
- 是否已上传成果
- 是否逾期
- 是否风险
- 是否待审核

总控可操作：
- 新增任务
- 修改任务
- 催办任务
- 标记风险
- 审核成果
- 查看成果

- 项目图片数量

字段：

- 未开始

- 待审核
- 已完成
- 已定稿
- 有风险

功能：
- 查看招商任务。
- 更新招商进度。

- 查看会员剩余权益。
- 标记客户确认参加。

同一图片素材只有一个主卡片，默认展示最新版。

上传新版本流程：
1. 点击「上传新版本」。
2. 选择新图片。
3. 弹出确认。
4. 确认后版本号递增。
5. 旧版本进入版本记录。

同名图片上传时提示：
- 更新为新版本（推荐）
- 作为新图片上传
- 取消

## 11. 签到管理

### 11.1 签到二维码

每个项目生成唯一签到二维码。

用途：
- 现场签到。
- 自动记录到场。
- 匹配人物资料。
- 判断是否有可用权益。
- 生成待客服核销记录。

### 11.2 老客户签到

扫码后填写或识别：
- 姓名
- 手机号

系统用手机号匹配人物资料。

如果匹配成功：
- 记录本场签到。
- 更新项目到场名单。
- 如果有可用权益，生成待客服确认核销。
- 招商中心显示已签到。
- 人物详情增加参与记录。

### 11.3 新客户临时建档

如果扫码人员不在资料库：

流程：
1. 填写姓名、手机号、公司/身份、来源渠道、推荐人/招商人员。
2. 系统查重。
3. 未命中则创建临时人物档案。
4. 签到成功。
5. 进入新客待跟进。

临时档案字段：
- 姓名
- 手机号
- 来源项目
- 签到时间
- 档案状态：临时
- 是否待完善
- 归属招商
- 负责客服

### 11.4 防重复逻辑

- 手机号完全相同：提示已存在。
- 姓名相同但手机号不同：提示可能重复。
- 手机号为空：允许创建，但标记信息不完整。
- 同一项目重复签到：提示已签到。

## 12. AI 周例会报告

### 12.1 功能定位

AI 周例会报告位于项目总览台。

目标：
一键把上一周各岗位任务进展、成果上传、风险变化、招商数据、签到数据、权益兑付情况生成 HTML 可视化报表，用于周例会展示。

### 12.2 数据来源

- 各岗位任务完成情况

### 12.3 生成配置

入口按钮：

```text
生成周例会报告
```

配置项：
- 报告范围：当前项目 / 全部进行中项目
- 时间范围：上一周 / 本周 / 自定义日期
- 报告格式：管理层简版 / 部门执行版 / 周例会展示版
- 包含内容：
  - 招商数据
  - 岗位任务
  - 图片成果
  - 风险提醒
  - 权益兑付
  - 签到/到场数据

1. 项目概览

4. 任务进度
5. 图片成果展示
6. 权益兑付情况

### 12.5 AI 生成规则

- AI 只基于系统已有数据生成。
- 无数据部分标注“暂无记录”。
- 风险建议需要引用对应任务或人员。
- 报告生成后允许人工编辑。
- 支持保存历史周报。

### 12.6 报告操作

生成后支持：
- 预览
- 编辑摘要
- 导出 HTML
- 导出 PDF
- 复制链接
- 保存为历史周报

## 14. 视觉方向

整体是公司内部运营工具。

设计原则：
- 专业
- 清晰
- 适合日常使用
- 信息密度适中偏高
- 不做营销落地页
- 不使用大面积装饰插画
- 不使用花哨渐变

延续现有人物素材库视觉：
- 浅米色背景
- 白色面板
- 黑色文字
- 黄色强调色
- 圆角不要过大
- 卡片和面板分区清晰

重点表达：
- 当前项目
- 任务分发
- 岗位执行
- 招商确认名单
- 签到状态
- 权益兑付
- 图片归档
- 版本管理
- AI 周报
